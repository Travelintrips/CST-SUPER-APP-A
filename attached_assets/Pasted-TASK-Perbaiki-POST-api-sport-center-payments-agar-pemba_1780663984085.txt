TASK: Perbaiki POST /api/sport-center/payments agar pembayaran muncul di Finance BizPortal

Tujuan:
- Semua pembayaran Sport Center tercatat di:
  - sport_payments
  - sport_bookings.payment_status
  - Supabase (sport_center_bookings)
  - Accounting (accounting_entries + accounting_payments)
- Dashboard Finance → Payments menampilkan data yang valid
- KPI Live revenue_today di dashboard Sport Center update otomatis

Instruksi:

1. Database Migration:
- Jalankan idempoten:
  ALTER TABLE sport_payments ADD COLUMN IF NOT EXISTS source TEXT NOT NULL DEFAULT 'SPORT_CENTER';
- Jalankan otomatis saat server start

2. Endpoint: POST /api/sport-center/payments
- Input:
  - amount ATAU total_amount
  - payment_method ATAU method
  - payment_date opsional (YYYY-MM-DD)
- Backend Logic:
  a. Fetch booking terlebih dahulu → ambil company_id
  b. Generate payment_number setelah company_id valid
  c. Insert ke sport_payments:
     - booking_id, amount, payment_method, paid_at = COALESCE(payment_date, NOW())
     - source = 'SPORT_CENTER'
     - payment_status = 'paid'
     - payment_type = 'booking'
  d. Update sport_bookings.payment_status = 'paid'
  e. Supabase sync:
     - panggil syncBookingUpsert() fire-and-forget
     - update sport_center_bookings di Supabase
  f. Accounting:
     - postSportCenterBookingWithTax() / postSportCenterBooking() fire-and-forget
     - Debit Kas ← Credit Pendapatan Sport Center
     - company_id dari booking
     - source = 'sport_center_booking'
  g. Accounting Payments:
     - insertAccountingPaymentForSportCenter() → muncul di Finance → Payments
  h. Audit Log:
     - catat booking_id, amount, method, payment_number, source ke sport_audit_logs
  i. Error handling:
     - catch(err) → console.error lengkap, tidak silent

3. Dashboard / KPI Live:
- revenue_today query dari accounting_entries WHERE source='sport_center_booking'
- Total Booking konsisten karena syncBookingUpsert() dipanggil setelah setiap payment

4. Middleware:
- requireAdmin (hanya admin BizPortal dapat akses)

5. Testing:
- Buat booking → bayar
- Cek:
  - sport_payments record ada, source='SPORT_CENTER', payment_status='paid'
  - sport_bookings.payment_status='paid'
  - Supabase sport_center_bookings update
  - Accounting journal terbentuk
  - Accounting Payments muncul di dashboard Finance
  - sport_audit_logs terisi
  - KPI Live revenue_today dan Total Booking konsisten
