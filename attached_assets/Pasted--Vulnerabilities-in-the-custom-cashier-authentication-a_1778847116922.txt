
Vulnerabilities in the custom cashier authentication and order authorization flow.

Vulnerabilities to fix:

[High] POS cashier tokens are forgeable and order detail reads ignore ownership Sistem POS kasir memperlakukan token login seolah-olah aman, padahal token itu hanya teks yang di-encode biasa dan bisa diubah siapa pun. Dengan memalsukan token atau mengganti identitas kasir di dalamnya, penyerang dapat menyamar sebagai kasir lain dan mengakses order yang bukan miliknya.
makeCashierToken() di artifacts/api-server/src/routes/posKasir.ts:38-40 membuat token dengan Buffer.from(JSON.stringify({ id, email })).toString("base64"). Tidak ada tanda tangan, MAC, expiry, atau verifikasi integritas. Di sisi klien, artifacts/customer-portal/src/lib/kasirAuth.ts:10-53 hanya menyimpan token itu di localStorage dan mengirimkannya kembali sebagai bearer token.

Pada saat request masuk, requireCashierAuth() di artifacts/api-server/src/routes/posKasir.ts:15-36 cukup melakukan base64 decode + JSON.parse, lalu lookup pos_cashiers.id. Server tidak memverifikasi bahwa token dibuat oleh server, tidak membandingkan email dengan record database, dan tidak memiliki mekanisme anti-forgery apa pun. Karena pos_cashiers.id adalah serial integer (lib/db/src/schema/posKasir.ts:7-16), identitas kasir juga mudah ditebak atau diubah.

Masalah ini diperparah oleh IDOR pada GET /api/pos-kasir/orders/:id di artifacts/api-server/src/routes/posKasir.ts:250-259. Berbeda dengan route pay dan cancel, endpoint detail order ini tidak memeriksa order.cashierId === cashier.id. Setelah berhasil memalsukan atau mengganti token ke kasir lain yang berstatus approved, attacker dapat mengambil detail order dan item milik kasir lain hanya dengan menebak orderId.

Eksploitasi praktisnya mudah: login sebagai kasir yang sah, decode token base64 dari browser, ubah id ke kasir approved lain, encode ulang, lalu panggil /api/pos-kasir/orders/:id. Karena server hanya percaya isi token dan tidak memeriksa ownership order, attacker memperoleh data transaksi lintas kasir, dan pada route lain dapat bertindak atas nama kasir yang diimpersonasi. Files: artifacts/api-server/src/routes/posKasir.ts, artifacts/customer-portal/src/lib/kasirAuth.ts, lib/db/src/schema/posKasir.ts
