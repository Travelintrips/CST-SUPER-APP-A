LANGUAGE SELECTOR:
Tambahkan komponen “Language Selector” / pemilihan bahasa yang modern, responsif, aksesibel, dan berkinerja tinggi.

Tujuan:
- User bisa memilih bahasa website.
- Komponen harus cocok dengan desain navbar modern.
- Tidak merusak logo dan layout navbar.

Posisi:
- Letakkan Language Selector di area kanan navbar, dekat Admin/Dashboard/Cart.
- Pada mobile, masukkan Language Selector ke dalam hamburger menu/drawer.

Desain:
- Gunakan icon globe/language.
- Tampilkan label singkat seperti:
  ID / EN
- Saat diklik, tampil dropdown pilihan bahasa.
- Dropdown harus modern:
  background putih/glass
  border halus
  shadow soft
  rounded corner 12px–16px
  hover state lembut
  active state jelas

Aksesibilitas:
- Gunakan button, bukan div clickable.
- Tambahkan aria-label="Pilih bahasa".
- Dropdown bisa dibuka dengan keyboard.
- Bisa ditutup dengan Escape.
- Bisa ditutup saat klik di luar.
- Gunakan focus ring yang jelas.
- Pastikan kontras teks cukup.

Performa:
- Jangan gunakan library berat jika tidak perlu.
- Gunakan data array lokal.
- Gunakan memoization jika daftar besar.
- Jangan render ulang berlebihan.

Fungsi:
- Simpan bahasa terpilih ke localStorage.
- Saat website dibuka ulang, bahasa terakhir tetap aktif.
- Jika sudah ada sistem i18n, integrasikan dengan sistem tersebut.
- Jika belum ada i18n, siapkan struktur data agar mudah dihubungkan nanti.

Data bahasa:
Gunakan daftar negara/lokasi berikut sebagai basis data, lalu mapping setiap negara ke bahasa utama/resmi yang paling relevan.

Komponen harus menampilkan:
- Flag
- Nama bahasa
- Negara/lokasi
- Kode bahasa singkat

Contoh tampilan dropdown:
🇮🇩 Bahasa Indonesia — Indonesia
🇺🇸 English — United States
🇨🇳 中文 — China
🇯🇵 日本語 — Japan

Default:
- Default bahasa: Bahasa Indonesia / id-ID.
- Jika browser language cocok dengan salah satu locale, boleh gunakan browser language sebagai default.
- Jika tidak cocok, gunakan id-ID.

Pastikan:
- Language Selector responsif di desktop dan mobile.
- Tidak merusak navbar.
- Tidak mengubah logo.
- Tidak membuat layout overflow.
- Tidak membuat error di console.
