Lanjutkan flow BizPortal setelah admin memilih vendor dan mengirim penawaran ke customer.

PENTING:
Gunakan MINI FORM TERPISAH.
Jangan masukkan flow customer approval ke dashboard utama.
Customer harus membuka link ringan dari WhatsApp/email.

Flow:
Admin sudah memilih vendor terbaik dari Admin Vendor Comparison.
Setelah itu admin klik tombol:
"Kirim Penawaran ke Customer"

Saat tombol diklik:
1. Admin menentukan harga jual final ke customer
2. Admin dapat mengisi ETA final
3. Admin dapat mengisi terms & conditions
4. Admin dapat mengisi catatan penawaran
5. Sistem membuat token unik untuk customer
6. Sistem membuat link:
   /customer-quote/:token
7. Sistem mengirim notifikasi WhatsApp/email ke customer
8. Status RFQ berubah menjadi customer_quoted

Customer Mini Form:
Route:
 /customer-quote/:token

Halaman harus mobile-friendly, ringan, dan bisa dibuka dari WhatsApp.

Customer hanya boleh melihat:
- RFQ number
- service type
- origin
- destination
- cargo detail
- harga final customer
- ETA final
- terms & conditions
- valid until
- catatan admin
- tombol Setuju
- tombol Minta Revisi
- tombol Tolak

Customer TIDAK boleh melihat:
- nama vendor
- harga vendor
- harga basic
- margin admin
- list comparison vendor
- data internal admin

Customer Action:

1. Jika customer klik SETUJU:
- tampilkan konfirmasi sebelum submit
- ubah status RFQ menjadi customer_approved
- buat order otomatis
- status order = order_confirmed
- simpan order_id ke RFQ
- kirim notifikasi ke admin
- kirim notifikasi ke vendor terpilih
- tampilkan halaman sukses ke customer

2. Jika customer klik MINTA REVISI:
- customer wajib isi catatan revisi
- ubah status RFQ menjadi customer_revision_requested
- simpan catatan revisi
- kirim notifikasi ke admin
- jangan buat order
- admin dapat revisi harga/ETA/terms lalu kirim ulang ke customer

3. Jika customer klik TOLAK:
- customer boleh isi alasan penolakan
- ubah status RFQ menjadi customer_rejected
- simpan alasan penolakan
- kirim notifikasi ke admin
- jangan buat order

Order Creation:
Saat customer approve, buat order baru otomatis.

Tabel orders harus menyimpan:

- final_customer_price

- updated_at

order_confirmed

Setelah order dibuat, lanjutkan ke Order Execution.

 /admin/orders/:order_id

- order number

- origin
- destination

- vendor terpilih

Admin dapat melakukan:
- assign vendor

- assign internal staff
- update status order

- kirim update ke customer
- buat invoice
- mark as completed

- update delivered

- waiting_pickup
- picked_up
- in_progress
- delivered
- pod_uploaded
- invoice_created
- payment_pending

Security:
- Customer quote hanya bisa dibuka dengan token valid
- Token expired tidak bisa approve
- Jika customer sudah approve/reject, link tidak boleh submit ulang
- Vendor cost dan margin tidak boleh tampil ke customer
- Internal notes tidak boleh tampil ke customer
- Order task hanya bisa dibuka oleh vendor/driver dengan token valid

Pastikan semua perubahan tersinkron ke dashboard utama BizPortal.
Dashboard utama hanya untuk admin monitoring, management, finance, reporting, dan settings.
Mini form harus tetap terpisah, cepat, ringan, dan mobile-first.
