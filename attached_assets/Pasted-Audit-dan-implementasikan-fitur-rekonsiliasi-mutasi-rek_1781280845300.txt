Audit dan implementasikan fitur rekonsiliasi mutasi rekening dari Google Sheet / upload Excel.

Tujuan:
App harus bisa mencocokkan mutasi bank dengan transaksi/order/payment di database berdasarkan:
1. tanggal transaksi
2. nominal masuk/keluar
3. tipe transaksi IN/OUT
4. keterangan mutasi
5. nama customer/vendor/tenant
6. khusus transaksi “DOMPET ANAK BANGSA / GoPay”, cocokkan juga berdasarkan Order ID / ID transaksi yang muncul di keterangan dan bukti transfer.

Format key utama mutasi:
bank_recon_key =
YYYYMMDD + "_" + amount + "_" + direction

direction:
- CR / credit / uang masuk = IN
- DB / debet / uang keluar = OUT

Contoh:
20260502_7150000_IN

Tambahkan juga normalized_description:
- lowercase
- hapus tanda baca
- trim spasi ganda
- ambil nama dari keterangan
- ambil order id jika ada pola seperti ID261222634814704JM

Matching rules:
1. Exact match:
   - tanggal sama
   - nominal sama
   - direction sama
   - order_id sama jika tersedia
   - otomatis match jika score >= 95

2. Strong match:
   - nominal sama
   - tanggal beda maksimal 1-3 hari
   - nama customer/vendor/tenant muncul di keterangan
   - score 80-94
   - masuk ke review admin

3. Duplicate:
   - jika mutation_key sama lebih dari 1
   - jangan auto approve
   - status duplicate_need_review
   - admin harus pilih transaksi mana yang benar

4. GoPay / DOMPET ANAK BANGSA:
   - extract Order ID dari keterangan, contoh ID261222634814704JM
   - cocokkan ke payment_gateway_ref, external_order_id, payment_reference, atau uploaded proof OCR text
   - jika order id sama dan nominal sama, auto match

5. Upload bukti transfer:
   - payment/order bisa punya uploaded_proof_url
   - tambahkan OCR text / extracted_reference bila tersedia
   - cocokkan keterangan mutasi dengan nama pengirim atau order id di bukti transfer
   - jika bukti transfer mengandung nominal/tanggal/order id yang sama, naikkan score

Backend:
- endpoint import mutasi dari Google Sheet / CSV / Excel
- endpoint POST /api/bank-reconciliation/import
- endpoint GET /api/bank-reconciliation/mutations
- endpoint GET /api/bank-reconciliation/matches/:mutationId
- endpoint POST /api/bank-reconciliation/:mutationId/approve
- endpoint POST /api/bank-reconciliation/:mutationId/reject
- endpoint POST /api/bank-reconciliation/run-matching

date + amount + direction + normalized description + customer/vendor name + provider_order_id + uploaded proof reference.
