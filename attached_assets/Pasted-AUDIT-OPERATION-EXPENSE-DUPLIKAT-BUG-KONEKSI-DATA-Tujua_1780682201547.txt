A. BACKEND AUDIT

1. **Duplicate Check**

   * Cek tabel expense (`expenses`, `kasbon`, `dana_talangan`, `cicilan_vendor`) untuk:

     * Record dengan `id` sama lebih dari satu
     * Record dengan kombinasi key unik (`logisticOrderId + category_id + source_account_id`)
   * Laporkan record duplikat

2. **Field Validation**

   * Semua record harus punya:

     * `category_id` â†’ kategori valid
     * `source_account_id` â†’ link ke COA Bank/Kas
     * `user_id` â†’ untuk kasbon karyawan
     * `vendor_id` / `recipient_id` â†’ untuk Dana Talangan / Cicilan Vendor
     * `amount` / `tax` terisi dan sesuai format
   * Tampilkan record dengan field kosong / invalid

3. **Link / Relasi Check**

   * Pastikan semua foreign key (`category_id`, `source_account_id`, `user_id`, `vendor_id`) ada di master table masing-masing
   * Pastikan invoice / POD / dokumen terkait (`invoice_id`, `receipt_url`) ada dan bisa diakses

4. **Audit Logs**

   * Pastikan setiap event expense (`created`, `approved`, `rejected`, `receipt_uploaded`, `payment_processed`) tercatat di `erp_audit_logs`
   * Tampilkan record tanpa log atau log gagal
