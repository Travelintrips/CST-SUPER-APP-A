Language Selector sudah mengubah sebagian teks, tetapi belum bekerja ke seluruh halaman. Tolong perbaiki agar fitur bahasa benar-benar aktif secara global di semua halaman dan terjemahannya akurat.

TARGET UTAMA:
1. Pilihan bahasa harus aktif di seluruh aplikasi, bukan hanya navbar.
2. Semua halaman harus memakai sistem translation yang sama.
3. Saat user pindah halaman, bahasa tetap sama.
4. Saat refresh browser, bahasa tetap sama.
5. Terjemahan harus rapi, natural, dan sesuai konteks bisnis logistik.
6. Jangan mengubah bentuk logo, desain logo, proporsi logo, atau file logo.

IMPLEMENTASI GLOBAL:
Jika project React:
1. Buat global i18n provider:
   src/i18n/LanguageProvider.jsx

2. Buat file:
   src/i18n/translations.js

3. Buat hook:
   useLanguage()

4. Semua komponen harus menggunakan:
   const { t, language, setLanguage } = useLanguage();

5. Semua teks hardcoded harus diganti menjadi:
   {t("translation_key")}

6. Jangan ada teks UI utama yang masih hardcoded.

7. Bungkus App dengan:
   <LanguageProvider>
     <App />
   </LanguageProvider>

8. Language Selector harus memanggil:
   setLanguage(locale)

9. Locale disimpan ke localStorage dengan key:
   "app_language"

10. Saat aplikasi dibuka:
    - cek localStorage
    - kalau ada, pakai itu
    - kalau tidak ada, cek navigator.language
    - kalau cocok dengan supported locale, pakai itu
    - kalau tidak cocok, default ke "id-ID"

FALLBACK:
- Jika translation untuk locale tertentu belum lengkap, fallback ke en-US.
- Jika en-US juga tidak ada, fallback ke id-ID.
- Jangan tampilkan translation key mentah ke user.
- Buat helper t(key, fallbackText) agar aman.

SUPPORTED LANGUAGE:
Aktifkan minimal:
- id-ID: Bahasa Indonesia
- en-US: English United States
- en-GB: English United Kingdom

Untuk bahasa lain di dropdown seperti China/Japan/Korea:
- Jika belum bisa diterjemahkan penuh, tetap boleh tampil di dropdown.
- Tetapi saat dipilih, gunakan fallback English sementara.
- Jangan sampai UI rusak atau key mentah muncul.

FORMAT TRANSLATION:
Gunakan nested object seperti:

export const translations = {
  "id-ID": {
    nav: {
      home: "Beranda",
      products: "Produk"
    }
  },
  "en-US": {
    nav: {
      home: "Home",
      products: "Products"
    }
  }
};

Function t harus mendukung dot notation:
t("nav.home")
t("about.title")

LANGUAGE SELECTOR:
1. Saat memilih bahasa, semua halaman langsung update tanpa reload.
2. Label tombol berubah sesuai bahasa aktif:
   ID untuk id-ID
   EN untuk en-US/en-GB
3. Checkmark aktif sesuai pilihan.
4. Dropdown tetap aksesibel.
5. Jangan merusak desain navbar.

QUALITY CHECK:
Setelah implementasi:
1. Klik Indonesia: seluruh halaman tampil Bahasa Indonesia.
2. Klik English US: seluruh halaman tampil English.
3. Klik English UK: seluruh halaman tampil English UK.
4. Pindah halaman: bahasa tetap sama.
5. Refresh: bahasa tetap sama.
6. Cek console: tidak ada error.
7. Cari semua teks hardcoded di komponen dan ganti ke t().
8. Pastikan tidak ada key seperti "about.title" yang tampil ke user.

PENTING:
Jangan hanya menerjemahkan navbar dan hero.
Audit semua file komponen dan halaman satu per satu.
Semua teks yang terlihat user harus masuk ke translations.js.
