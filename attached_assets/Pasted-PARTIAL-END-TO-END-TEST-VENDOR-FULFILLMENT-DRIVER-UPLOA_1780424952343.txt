PARTIAL END-TO-END TEST — VENDOR FULFILLMENT → DRIVER UPLOAD → CUSTOMER TIMELINE

Tujuan:
Test seluruh alur dari tahap vendor submit fulfillment atau customer approve sampai driver upload gambar dan tampil di customer tracking page, secara partial dan terkontrol.

---

A. BACKEND SETUP

1. **Siapkan dummy data**

   * orderId, jobId, logisticOrderId valid di staging DB
   * driverId / driver_phone (dummy) jika ingin test driver upload

2. **Vendor Fulfillment Submit**

   * Endpoint: `POST /api/logistic/orders/:orderId/fulfillment`
   * Payload:

     ```
     {
       driverId: "DUMMY_DRIVER_ID",
       jobId: "DUMMY_JOB_ID",
       logisticOrderId: "DUMMY_ORDER_ID",
       fulfillmentType: "Trucking"
     }
     ```
   * Simpan ke DB (`driver_jobs` / `fulfillment_jobs`)
   * Trigger timeline update:

     * `updateOrderProgress(logisticOrderId)`
     * SSE / WebSocket broadcast (opsional)
   * Trigger WA / push notif:

     * Admin → fulfillment submitted
     * Customer → dummy log, WA tidak dikirim live

3. **Customer Approve Order (opsional)**

   * Endpoint: `POST /api/orders/:orderId/approve`
   * Update order status → timeline update
   * Trigger SSE / WebSocket → timeline refresh
   * Trigger WA / push → log debug saja

4. **Driver Upload POD / Image**

   * Endpoint: `POST /api/driver/jobs/:jobId/pod`
   * Payload:

     ```
     {
       driverId: "DUMMY_DRIVER_ID",
       jobId: "DUMMY_JOB_ID",
       logisticOrderId: "DUMMY_ORDER_ID",
       file: [upload file path]
     }
     ```
   * Simpan file → storage → simpan URL / path di DB
   * Relasi ke jobId & logisticOrderId
   * Trigger timeline update (`updateOrderProgress`) & SSE / WebSocket
   * Trigger WA / push → log debug (opsional)

5. **Logging**

   * Fulfillment type
   * Driver phone / driverId
   * File URL
   * Timeline update status
   * WA send result (log, jangan kirim live)

---

B. FRONTEND SETUP

1. **Customer Tracking Page**

   * GET `/api/order/:logisticOrderId/progress`
   * Render timeline:

     * Step fulfillment submitted
     * Step approved
     * Step POD uploaded / Bukti Pengiriman
   * Render attachment thumbnail / preview
   * Expandable timeline → klik untuk detail

2. **Timeline / Status Card**

   * Badge / dot color coding:

     * Grey = belum
     * Green = selesai
     * Indigo = aktif
     * Red = cancelled
   * Realtime update via SSE / polling
