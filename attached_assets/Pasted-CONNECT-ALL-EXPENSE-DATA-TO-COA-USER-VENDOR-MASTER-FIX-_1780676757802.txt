CONNECT ALL EXPENSE DATA TO COA, USER & VENDOR MASTER — FIX

Tujuan:
Pastikan semua expense di sistem terhubung dengan data master, termasuk:

* COA Bank / Kas untuk sumber dana
* User / Karyawan untuk kasbon karyawan
* Vendor / Pihak penerima untuk Dana Talangan / Pembayaran Vendor
* Kategori expense / preset category
* Mapping otomatis ke jurnal DR/CR

---

A. BACKEND

1. **Update Expense Table**

   * Pastikan field:

     * `source_account_id` → link ke COA Bank / Kas
     * `category_id` → link ke kategori expense
     * `user_id` → untuk kasbon karyawan
     * `vendor_id` / `recipient_id` → untuk Dana Talangan / Pembayaran Vendor

2. **Endpoints GET**

   * `/api/expenses`, `/api/kasbon`, `/api/dana-talangan`
   * Sertakan full mapping:

     ```json
     {
       "expense_id": "...",
       "category": {"id":"...","name":"..."},
       "source_account": {"id":"...","name":"Bank BCA","type":"bank"},
       "user": {"id":"...","name":"John Doe"},
       "vendor": {"id":"...","name":"PT Supplier"},
       "amount": 100000,
       "tax": "PPN 11%"
     }
     ```

3. **Validation / Auto-fill**

   * Saat buat expense:

     * Dropdown Sumber Dana → fetch dari COA (`type = bank/kas`)
     * Dropdown Karyawan → fetch user master
     * Dropdown Vendor / Pihak Penerima → fetch vendor master
     * Dropdown Kategori → fetch kategori preset
   * Auto-fill default jika kategori punya default akun / pajak

---

B. FRONTEND

1. **Form Expense**

   * Dropdown auto-fill:

     * Sumber Dana = `coa_id`
     * Karyawan = `user_id`
     * Vendor = `vendor_id`
     * Kategori = `category_id`
   * Selected value dikirim ke backend untuk mapping jurnal

2. **Expense List**

   * Kolom:

     * Sumber Dana → `source_account.name`
     * Karyawan → `user.name`
     * Vendor / Pihak Penerima → `vendor.name`
     * Kategori → `category.name`
   * Pastikan semua kolom tampil lengkap
