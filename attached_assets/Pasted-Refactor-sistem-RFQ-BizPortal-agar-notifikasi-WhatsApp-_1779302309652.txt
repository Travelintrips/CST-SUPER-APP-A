Refactor sistem RFQ BizPortal agar notifikasi WhatsApp admin berbentuk recap list vendor terbaru.

Tujuan:
Saat vendor submit atau update penawaran, sistem jangan membuat form comparison baru.
Gunakan 1 halaman comparison yang sama untuk setiap RFQ:
 /admin-vendor-comparison/:rfq_id

Flow utama:
1. Customer membuat request.
2. Request masuk ke Admin sebagai RFQ.
3. Admin memilih beberapa vendor.
4. Admin klik Blast ke Vendor.
5. Sistem membuat token unik untuk setiap vendor.
6. Vendor membuka /vendor-form/:token.
7. Vendor dapat:
   - Terima harga basic
   - Ajukan harga baru
   - Tolak
8. Semua jawaban vendor masuk ke RFQ yang sama.
9. Admin melihat semua jawaban di satu halaman:
   /admin-vendor-comparison/:rfq_id
10. Admin memilih vendor terbaik.
11. Admin menentukan harga jual ke customer.
12. Customer hanya melihat harga final, bukan harga vendor.

Ubah flow lama:
- Nonaktifkan logic customer blast langsung ke vendor.
- Semua RFQ baru wajib melalui admin terlebih dahulu.
- RFQ lama tetap bisa dibaca, tetapi RFQ baru harus memakai flow Customer → Admin → Vendor → Admin → Customer.

Vendor Form:
Route:
 /vendor-form/:token

Vendor form harus menampilkan:
- RFQ number
- service type
- origin
- destination
- cargo detail
- harga basic / target price
- deadline response

Vendor bisa memilih:
1. Terima Harga Basic
2. Ajukan Harga Baru
3. Tolak

Jika Terima Harga Basic:
- vendor_response_price = basic_price
- status = accepted_basic_price

Jika Ajukan Harga Baru:
- wajib isi harga baru
- wajib isi ETA
- wajib isi catatan/alasan
- status = counter_offer

Jika Tolak:
- wajib isi alasan optional
- status = rejected

Admin Comparison Page:
Route:
 /admin-vendor-comparison/:rfq_id

Halaman ini harus menjadi single live comparison page.
Untuk 1 RFQ hanya boleh ada 1 halaman comparison.
Jangan buat halaman/form baru setiap vendor submit.

Tampilkan:
- RFQ number
- customer name
- service type
- origin
- destination
- deadline
- total vendor dikirim
- vendor sudah jawab
- vendor belum jawab
- vendor reject

- vendor expired

Tampilkan daftar vendor:
- nama vendor
- harga basic
- harga penawaran vendor
- ETA
- status
- catatan
- attachment quotation
- waktu submit/update
- label NEW jika baru update
- tombol Pilih Vendor Ini
- tombol Minta Revisi
- tombol Tolak

Status vendor:
- waiting_response
- accepted_basic_price
- counter_offer
- rejected
- expired
- selected
- not_selected
- late_response

Jika vendor belum menjawab sampai deadline:
- ubah status menjadi expired

Jika admin sudah memilih vendor lalu vendor lain submit terlambat:
- simpan sebagai late_response
- jangan ubah vendor terpilih otomatis

Realtime / Auto Refresh:
- Halaman admin comparison harus auto-refresh setiap 5-10 detik
- Jika memungkinkan gunakan realtime subscription
- Vendor response terbaru harus muncul tanpa reload manual
- Vendor response terbaru pindah ke atas
- Tampilkan badge merah untuk update baru

- Tambahkan activity timeline

Activity Timeline:
- vendor submit

- vendor reject

- admin select vendor

WhatsApp Notification Admin:
Saat vendor submit atau update penawaran:
1. simpan data ke database
2. ambil semua vendor response untuk RFQ yang sama
3. ambil juga vendor yang belum menjawab
4. generate recap list terbaru
5. kirim WhatsApp ke admin
6. semua notifikasi harus mengarah ke link comparison yang sama

Layanan: {service_type}
Rute: {origin} → {destination}

Daftar penawaran:

2. {vendor_name} — Rp{price} — ETA {eta} — {status}
3. {vendor_name} — Rp{price} — ETA {eta} — {status}

Belum jawab:
- {vendor_name}
- {vendor_name}

Lihat & pilih vendor:
{admin_comparison_link}
