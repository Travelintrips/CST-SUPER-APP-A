FASE 2B — CREATE SERVICE_TEMPLATES DB TABLE

Tujuan:
Membuat tabel service_templates sebagai DB override untuk lib/service-templates, tanpa mengubah behavior aplikasi.

Kondisi:
- FASE 2A selesai.
- lib/service-templates sudah ada.
- 17 in-code service templates sudah tersedia.
- SERVICE_SCHEMAS masih berjalan normal.

Tugas:

1. Buat Drizzle schema baru:

lib/db/src/schema/serviceTemplates.ts

Table:
service_templates

Columns:
- id serial primary key
- serviceType text unique not null
- label text not null
- emoji text default '📋'
- version text default '1.0.0'
- isActive boolean default true
- description text nullable
- sortOrder integer default 0
- fields jsonb default []
- requiredDocuments jsonb default []
- checklist jsonb default []
- conditionalRules jsonb default []
- validationRules jsonb default []
- createdAt timestamp default now
- updatedAt timestamp default now

2. Export schema dari:
lib/db/src/schema/index.ts

3. Buat boot migration idempotent:

CREATE TABLE IF NOT EXISTS service_templates ...

4. Buat seed script idempotent:

scripts/seed-service-templates.mjs

Seed dari:
@workspace/service-templates

Masukkan semua 17 template:
- product

- air_freight
- ppjk

- customer_shipment
- customer_quote
- customer_document

- pertahankan id
