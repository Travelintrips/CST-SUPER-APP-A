Tolong revisi dan upgrade module Accounting / Finance pada aplikasi saya menjadi MULTI COMPANY ACCOUNTING SYSTEM.

Saya memiliki 4 perusahaan:

1. PT Cahaya Sejati Teknologi
2. PT Wangsamas
3. PT Diva Servis
4. PT Elmira Ratu Abadi

Yang saya inginkan:
Satu aplikasi tetapi bisa mengelola accounting masing-masing perusahaan secara terpisah dan professional seperti ERP/accounting enterprise.

FITUR YANG HARUS DIBUAT:

1. Multi Company Structure
- Tambahkan master table companies
- Setiap transaksi accounting wajib memiliki company_id
- Semua data accounting harus terpisah per perusahaan:
  - COA
  - Journal
  - Invoice
  - Expense
  - Revenue
  - Bank Account
  - Cashflow
  - Asset
  - Tax
  - Vendor
  - Customer
  - Balance Sheet
  - Profit Loss
  - Trial Balance

2. Company Switcher
Buat company switcher di navbar/header.

Contoh:
[ PT Cahaya Sejati Teknologi ▼ ]

Ketika user memilih perusahaan:
- seluruh dashboard berubah mengikuti company tersebut
- seluruh laporan otomatis filter by company_id
- session terakhir tersimpan

3. Database Structure
Tambahkan table:

companies
- id
- company_name
- company_code
- logo_url
- address
- phone
- email
- npwp
- created_at

Tambahkan company_id pada:
- journals
- invoices
- expenses
- revenues
- bank_accounts
- coa_accounts
- transactions
- payments
- vendors
- customers
- assets
- tax_reports
- quotation
- orders

4. Separate COA per Company
Setiap perusahaan bisa memiliki:
- chart of accounts berbeda
- nomor akun berbeda
- bank account berbeda
- saldo berbeda

Jangan share saldo antar perusahaan.

5. Dashboard Accounting
Dashboard harus menampilkan:
- Total Revenue
- Total Expense
- Profit/Loss
- Cashflow
- Outstanding Invoice
- Bank Balance

berdasarkan perusahaan aktif.

6. Financial Reports
Semua laporan wajib bisa:
- filter per company
- export PDF
- export Excel

Laporan:
- Profit & Loss
- Balance Sheet
- General Ledger
- Trial Balance
- Cashflow
- Journal Report
- Tax Report

7. User Access / Permission
Tambahkan role:
- Super Admin
- Company Admin
- Finance Staff

Super Admin:
- bisa lihat semua perusahaan

Company Admin:
- hanya bisa melihat company tertentu

Finance Staff:
- hanya transaksi tertentu

8. Transaction Isolation
PENTING:
Data antar perusahaan tidak boleh tercampur.

Contoh:
Invoice PT Wangsamas tidak boleh muncul di PT Diva Servis.

Semua query wajib menggunakan:
WHERE company_id = activeCompanyId

9. Auto Company Tagging
Saat user membuat:
- invoice
- journal
- expense
- quotation
- order

system otomatis mengisi:
company_id = perusahaan aktif saat ini.

10. Company Branding
Setiap perusahaan bisa memiliki:
- logo sendiri
- invoice template sendiri
- quotation header sendiri
- alamat & NPWP sendiri

11. Accounting Flow
Pastikan:
- journal posting tetap balance
- debit/credit tetap berjalan normal
- multi company tidak merusak sistem existing

12. Migration Existing Data
Buat migration script:
- assign existing data ke PT Cahaya Sejati Teknologi sebagai default company
- jangan merusak data existing

13. UI / UX
Buat UI terlihat professional seperti:
- ERPNext
- Odoo
- Jurnal.id
- Accurate
- SAP Business One

Gunakan:
- company badge
- company selector
- company logo
- company colored theme optional

14. Security
Pastikan:
- user tidak bisa akses company lain tanpa permission
- API wajib validate company_id
- semua dashboard aman multi tenant

15. Future Ready
Structure harus siap untuk:
- tambah perusahaan baru di masa depan
- intercompany transaction
- consolidated report
- multi branch
- multi warehouse

Catatan penting:
Saya ingin sistem multi company enterprise yang scalable dan clean architecture, bukan hanya filter sederhana di frontend.

Gunakan:
- centralized multi-tenant architecture
- backend validation
- company context provider
- reusable company middleware
- proper database relation

Pastikan semua halaman accounting sudah support multi company.
