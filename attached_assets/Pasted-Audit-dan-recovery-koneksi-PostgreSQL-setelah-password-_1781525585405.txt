Audit dan recovery koneksi PostgreSQL setelah password diperbaiki.

Situasi:

* Password database sudah diperbaiki.
* Circuit Breaker masih menunjukkan:
  "too many authentication failures, new connections are temporarily blocked"
* Perlu memastikan apakah blokir hanya sisa cooldown atau masih ada service yang memakai credential lama.

Kerjakan langkah berikut secara berurutan:

1. Tunggu cooldown selesai atau lakukan:
   POST /api/system/reset-circuit-breaker

2. Setelah reset, jalankan:
   GET /api/system/db-connections

Tampilkan:

* circuitBreaker.open
* remainingCooldownSeconds
* poolStats
* activeDb.source
* activeDb.host
* activeDb.mode

3. Tambahkan endpoint:
   GET /api/system/db-auth-check

Response:

{
"host": "...",
"port": 6543,
"username": "...",
"database": "...",
"pooler": true,
"canConnect": true,
"authError": null,
"connectionSource": "SUPABASE_DATABASE_URL"
}

Jangan tampilkan password.

4. Jalankan test koneksi langsung ke PostgreSQL menggunakan credential aktif.
   Pastikan:

* login berhasil
* SELECT 1 berhasil
* tidak ada auth error

5. Cari seluruh penggunaan:

* DATABASE_URL
* PROD_DATABASE_URL
* SUPABASE_DATABASE_URL

di runtime code.

Tampilkan:

* file
* env yang digunakan
* host target
* apakah berbeda dari pool utama

6. Jika ditemukan credential berbeda:

* tampilkan file
* tampilkan env yang dipakai
* perbaiki agar memakai sumber yang sama dengan pool utama

7. Setelah koneksi DB dipastikan normal:

* klik Refresh
* klik Resync Fasilitas
* klik Resync Booking
* klik Sync Akuntansi

8. Catat hasil:

* jumlah sukses
* jumlah gagal
* error detail jika ada

9. Jika masih gagal:
   ambil dari Audit CB:

* rootCause
* firstEcbOpeningError
* errorsByCategory
* lastTrigger
* lastSyncErrors

Output akhir yang saya mau:

A. Status DB:

* canConnect
* username
* host
* source env

B. Status Circuit Breaker:

* open/closed
* cooldown

C. Audit credential:

* file yang masih memakai env lama
* file yang sudah benar

D. Hasil sync:

* fasilitas
* booking
* payment
* accounting

E. Root cause final jika masih gagal.
