Tolong upgrade fitur Vendor Mini Form menjadi sistem mini form transaksional sesuai tipe layanan/item, bukan hanya form rate umum.

ALUR UTAMA:
Customer Request Order → Admin Review/Approve → Admin kirim link Mini Form ke vendor sesuai tipe layanan → Vendor isi approval/penawaran → Admin review → Admin kirim ke Customer → Customer setuju → sistem otomatis buat SO → Admin kirim instruksi final ke Vendor → Vendor isi data operasional sesuai tipe layanan.

SERVICE TYPE MINI FORM:
Buat template form berbeda untuk:
1. Product
2. Trucking
3. Freight Sea
4. Freight Air
5. PPJK / Customs Clearance
6. Handling / Warehouse
7. Document / Additional Service

KONSEP LINK:
Setiap link Mini Form harus terhubung ke:
- order_id
- order_item_id
- service_type
- vendor_id atau vendor_name
- status link
- expired_at optional
- unique_token

Jangan hanya generate link umum. Link harus spesifik per item agar vendor hanya melihat kebutuhan yang relevan.

CONTOH:
Jika order customer berisi:
- Produk
- Trucking
- PPJK

Maka admin bisa generate 3 link berbeda:
- Mini Form Produk untuk Vendor A
- Mini Form Trucking untuk Vendor B
- Mini Form PPJK untuk Vendor C

FITUR ADMIN:
Di halaman Mini Form / Order Detail, admin harus bisa:
1. Pilih order item
2. Pilih vendor
3. Generate link Mini Form sesuai service_type
4. Kirim link ke WhatsApp vendor
5. Lihat semua submission vendor per item
6. Bandingkan harga vendor
7. Pilih vendor terbaik
8. Kirim penawaran final ke customer
9. Jika customer approve, otomatis buat Sales Order
10. Kirim link final confirmation ke vendor untuk isi data operasional

FITUR CUSTOMER APPROVAL:
Buat mini form customer approval publik:
- Customer melihat ringkasan penawaran
- Harga jual
- Service detail
- Terms & conditions
- Button Setuju
- Button Tolak
Jika customer klik Setuju:
- status menjadi customer_approved
- sistem otomatis create SO
- admin mendapat notifikasi
- vendor terpilih bisa dikirim instruksi final

Terhubung ke order_id dan order_item_id untuk transaksi real.
