FASE 4 — IMPLEMENTASI SINKRONISASI END-TO-END

Tujuan:
Update driver progress (INTERNAL WA Mini Form / EXTERNAL Driver App) otomatis tercatat di driver_jobs, POD/Delivery Status, Order Timeline, dan notifikasi Admin/Customer.

---

A. BACKEND — PATCH / GET & Driver Progress

1. **PATCH /api/drivers/jobs/:jobId/status (Admin)**

   * Mendukung cancel + force update
   * Trigger WA ke Admin + Customer saat:

     * PICKED_UP
     * DELIVERED
     * COMPLETED
   * Return status log terbaru

2. **GET /api/drivers/jobs/list**

   * Sertakan `statusLogs` → untuk timeline di frontend
   * Filterable berdasarkan `logisticOrderId` dan `shipmentId`
   * Pastikan INTERNAL job (driverId null) termasuk

3. **Driver App PATCH / PUT status**

   * Trigger admin WA saat status PICKED_UP / DELIVERED
   * Update timeline & order status via updateOrderProgress / transitionLogisticOrderStatus
   * INTERNAL job juga harus update jika driverId null

4. **driverProgress.ts**

   * Tambahkan trigger admin WA untuk PICKUP / DELIVERED
   * Pastikan legacy driver_jobs tetap tampil dan tidak crash
   * Update orderProgress untuk INTERNAL job (driverId null)

---

B. FRONTEND — OrderDriverAssignmentPanel / Timeline

1. Tambahkan **Timeline component**:

   * Sequence: Assigned → Pickup Started → Picked Up → In Transit → Delivered → Completed
   * Badge/status update realtime via polling API `/drivers/jobs?logisticOrderId=xxx` atau Realtime subscription

2. Perbarui **cancel assignment button**:

   * Panggil PATCH admin route baru
   * Update status → timeline → trigger WA Admin

3. ActiveJobCard / panel assignment:

   * Tampilkan badge INTERNAL / EXTERNAL
   * Tampilkan WA Mini Form link (INTERNAL)
   * Tampilkan CST Driver App badge & akun status (EXTERNAL)
   * Timeline menampilkan semua `statusLogs`

3. Pastikan legacy driver_jobs dapat mengirim notif jika status update
