Saya memakai Fonnte untuk WhatsApp gateway. Tolong buat fitur Mini Form Reply untuk admin/vendor agar balasan quotation bisa dikirim ke customer via Fonnte.

Kebutuhan:

1. Cari dulu integrasi Fonnte yang sudah ada di project:
   - endpoint send WhatsApp
   - environment variable FONNTE_TOKEN
   - fungsi/service pengiriman pesan WhatsApp
   - route webhook Fonnte jika sudah ada

2. Jangan membuat integrasi WhatsApp baru dari nol jika Fonnte sudah ada. Gunakan service Fonnte existing.

3. Buat Mini Form dengan field:
   - RFQ ID / Order ID
   - Nama Customer
   - Nomor WhatsApp Customer
   - Nama Vendor
   - Nomor WhatsApp Vendor
   - Jenis Layanan
   - Rute
   - Harga Vendor
   - Markup Type: persen / nominal
   - Markup Value
   - Harga Final Customer
   - Estimasi Pickup
   - Estimasi Delivery
   - Catatan
   - Status: Ready / Need Info / Not Available

4. Harga Final Customer dihitung otomatis:
   - jika markup persen: harga vendor + persentase markup
   - jika markup nominal: harga vendor + nominal markup

5. Buat preview pesan WhatsApp sebelum dikirim.

Format pesan WhatsApp ke customer:

Halo {customerName},

Berikut quotation layanan CST Logistics:

No. RFQ: {rfqId}
Layanan: {serviceType}
Rute: {route}
Estimasi Pickup: {pickupDate}
Estimasi Delivery: {deliveryDate}
Harga Final: Rp {finalPrice}
Status: {status}

Catatan:
{notes}

Silakan konfirmasi apabila quotation ini disetujui.

Terima kasih,
CST Logistics

6. Tombol:
   - Save Draft
   - Preview WhatsApp
   - Send to Customer via Fonnte
   - Send to Admin Group via Fonnte

7. Untuk pengiriman Fonnte, gunakan endpoint:
   https://api.fonnte.com/send

Header:
   Authorization: process.env.FONNTE_TOKEN

Body minimal:
   target: nomor tujuan WhatsApp
   message: isi pesan

8. Buat API route backend:
   POST /api/whatsapp/fonnte/send-quotation

Payload:
   {
     rfqId,
     customerName,
     customerPhone,
     vendorName,
     vendorPhone,
     serviceType,
     route,
     vendorPrice,
     markupType,
     markupValue,
     finalPrice,
     pickupDate,
     deliveryDate,
     notes,
     status,
     sendToAdminGroup
   }

9. Simpan log ke database:
   - rfqId
   - customerPhone
   - vendorName
   - vendorPhone
   - vendorPrice
   - markupType
   - markupValue
   - finalPrice
   - messageBody
   - fonnteResponse
   - sentStatus
   - sentAt
   - createdAt

10. Pastikan nomor WA dinormalisasi ke format Indonesia, contoh:
   08123456789 menjadi 628123456789.

11. Jangan mengirim harga vendor ke customer. Customer hanya menerima harga final yang sudah markup.

12. Admin harus bisa review sebelum pesan dikirim ke customer.

13. Setelah selesai, jelaskan:
   - file yang dibuat/diubah
   - env yang diperlukan
   - cara testing kirim WA via Fonnte
   - cara cek log pengiriman
